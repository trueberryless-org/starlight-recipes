The Starlight Recipes plugin can be configured inside the `astro.config.mjs` configuration file of your project:

```js {11}
// astro.config.mjs
import starlight from '@astrojs/starlight'
import { defineConfig } from 'astro/config'
import starlightRecipes from 'starlight-recipes'

export default defineConfig({
  integrations: [
    starlight({
      plugins: [
        starlightRecipes({
          // Configuration options go here.
        }),
      ],
      title: 'My Recipes',
    }),
  ],
})
```

## Plugin configuration

The Starlight Recipes plugin accepts the following configuration options:

### `recipeCount`

**Type:** `number`<br />
**Default:** `5`

The number of recipes to display per page in the recipe list.

### `popularRecipeCount`

**Type:** `number`<br />
**Default:** `3`

The number of popular recipes to display in the sidebar.

Popularity is determined by sorting by the average rating value.
See the [rating system](/interactive/rating-system#how-the-rating-system-works) for details.

### `authors`

**Type:** [`StarlightRecipesAuthorsConfig`](#author-configuration)

A list of global authors for all recipes or regular authors that can be referenced in individual recipes.
Check the ["Authors" guide](/guides/authors) for more information.

### `prefix`

**Type:** `string`<br />
**Default:** `'recipes'`

The base prefix for all recipe routes.

By default, an overview of all recipes will be available at `/recipes` and individual recipes at e.g. `/recipes/virgin-colada`.
Setting this option to `'cookbook'` will make an overview of all recipes available at `/cookbook` and individual recipes at e.g. `/cookbook/virgin-colada`.

### `processVideo`

**Type:** `boolean`<br />
**Default:** `false`

Whether to automatically fetch and process metadata for YouTube videos provided in recipe [frontmatter](/guides/frontmatter) ([`video` property](/guides/frontmatter#video)).

When enabled, the plugin will attempt to fetch information such as the video title, description, thumbnail, and duration from YouTube.
This data is then used to populate the [`video` property](/guides/structured-data#video) in [Structured Data](/guides/structured-data), which is highly recommended by search engines for recipe visibility.

:::caution[Performance Warning]
Enabling this feature results in approximately a **threefold (3x) increase** in build duration, as the plugin must perform network requests for every recipe containing a video. Use this only if the SEO benefits outweigh the impact on your development and CI/CD workflow.
:::

### `cookingMode`

**Type:** [`CookingModeConfig`](#cookingmodeconfig)

Configuration for the interactive cooking experience.

:::note
No server adapters are needed for these options to work as they all run on the client side.
:::

#### `CookingModeConfig`

| Field | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `stepTimer` | `boolean` | `true` | Enables countdown timers for instructions. Requires a `time` field in the [instruction step frontmatter](/guides/frontmatter#instructionstep). |
| `stepCheckbox` | `boolean` | `true` | Toggles interactive checkboxes on step numbers to track progress for instructions. |

## Author configuration

Global authors for all recipes or regular authors that can be referenced in individual recipes can be defined using the [`authors`](#authors) configuration option.
When a recipe frontmatter does not define an author, the global authors from the configuration will be used instead.
Check the ["Authors" guide](/guides/authors) for more information.

```js {3-8}
starlightRecipes({
  authors: {
    alice: {
      // Author configuration for the `alice` author goes here.
    },
    bob: {
      // Author configuration for the `bob` author goes here.
    },
  },
})
```

An author can be configured using the following options:

### `name`

**Required**<br />
**Type:** `string`

The name of the author.

### `title`

**Type:** `string`

A title to display below the author's name.

### `url`

**Type:** `string`

A URL to link the author's name to.

### `picture`

**Type:** `string`

A URL or path to an image to display as the author's picture.

- For local images in `src/`, provide the relative path to the image, e.g. `./src/assets/author.png`.
- For local images in `public/`, provide an absolute path starting with a `/`, e.g. `/images/author.png`.
- For remote images, provide the URL to the image. When using remote images, check out the [“Authorizing remote images”](https://docs.astro.build/en/guides/images/#authorizing-remote-images) guide to enable image optimization.
