Starlight Recipes exposes information about the recipes and authors in your project.
This guide explains how to use Starlight Recipes’ data.

## What is recipes data?

Starlight Recipes data is an object containing the information about all the recipes and authors in your project.
It includes information like a list of all the recipe authors, a list of all the recipes, the title of each recipe, the date it was published, and the author.

## Using recipe data

The [`recipes`](#recipes) property contains an array of all the recipes in your project while the [`authors`](#authors-1) property contains an array of all the authors.

You can access this data from the `Astro.locals.starlightRecipes` global in Astro components:

```astro title="example.astro"
---
const { recipes, authors } = Astro.locals.starlightRecipes
//        │        └─ The list of all recipe authors
//        └─ The list of all recipes
---
```

### Example: Recipes

This can be useful for example to create a widget that lists popular recipes with their average rating on your homepage:

import Preview from '@/components/Preview.astro'
import RecipesDataExample from '@/components/RecipesDataExample.astro'

<Preview>

```astro title="PopularRecipes.astro" "Astro.locals.starlightRecipes.recipes"
<ul>
  {
    Astro.locals.starlightRecipes.recipes
      .sort((a, b) => {
        const ratingA = a.averageRating?.ratingValue ?? 0;
        const ratingB = b.averageRating?.ratingValue ?? 0;
        return ratingB - ratingA;
      })
      .slice(0, 5)
      .map((recipe) => (
        <li>
          <a href={recipe.href}>{recipe.title}</a>
          <em>({recipe.averageRating?.ratingValue}/5 stars)</em>
        </li>
      ))
  }
</ul>
```

const authorNames = Astro.locals.starlightRecipes.authors
  .map((author) => author.name)
  .sort((a, b) => a.localeCompare(b, Astro.currentLocale))

const authorList = new Intl.ListFormat(Astro.currentLocale, {
  style: 'long',
  type: 'conjunction',

## `starlightRecipes` object

The `starlightRecipes` object contains information about all recipes and authors.
Here is a summary of the whole type of the object:

```graphql
{
    recipes: {
        title: string;
        href: string;
        createdAt: Date;
        updatedAt: Date | undefined;
        draft: boolean;
        featured: boolean;
        cover: { alt: string; image: ImageMetadata | string } | { alt: string; dark: ImageMetadata | string; light: ImageMetadata | string } | undefined;
        entry: CollectionEntry;
        authors: { name: string; title: string | undefined; url: string | undefined; }[];
        tags: { label: string; href: string; }[];
    }[];
    authors: { name: string; title: string | undefined; url: string | undefined; }[];
}
```

Below you can find details about each property:

### `recipes`

A list of all the recipes in your project ordered by descending [publication date](/guides/frontmatter#date).

Each recipe has the following properties:

#### `title`

**Type:** `string`

The [title](/guides/frontmatter#title) of the recipe.

#### `href`

**Type:** `string`

The link to the recipe.

#### `createdAt`

**Type:** `Date`

The [date](/guides/frontmatter#date) of the recipe.

#### `updatedAt`

**Type:** `Date | undefined`

The last update date of the recipe.
It is only defined if the [`lastUpdated`](/guides/frontmatter#lastupdated) frontmatter field is specified and different from the [`date`](/guides/frontmatter#date) frontmatter field.

#### `draft`

**Type:** `boolean`

Whether the recipe is a [draft](/guides/frontmatter/#draft).
Draft recipes are only visible in development mode.

#### `featured`

**Type:** `boolean`

Whether the recipe is [featured](/guides/frontmatter/#featured).

#### `cover`

**Type:** `{ alt: string; image: ImageMetadata | string } | { alt: string; dark: ImageMetadata | string; light: ImageMetadata | string } | undefined`

The optional cover image of the recipe.

Based on how the cover image is [defined](/guides/frontmatter#cover) in the frontmatter of the recipe, it can either be a single image or a pair of images for dark and light themes.
Local images are imported and transformed into metadata while remote images are defined as a string URL that you can pass as a `src` to `<Image/>` or `getImage()`.

Check the Starlight Recipes [`<Cover>` component](https://github.com/trueberryless/starlight-recipes/blob/main/packages/starlight-recipes/components/metadata/Cover.astro) to see an example of how to use the cover image or learn more about [images in content collections](https://docs.astro.build/en/guides/images/#images-in-content-collections) in the Astro docs.

#### `entry`

The Astro content collection entry for the recipe which includes frontmatter values at `entry.data`.

```js
entry: {
  data: {
    description: string | undefined
    // etc.
  }
}
```

Learn more about the shape of this object in [Astro’s Collection Entry Type](https://docs.astro.build/en/reference/modules/astro-content/#collectionentry) reference.

#### `authors`

**Type:** <code><a href="#starlightrecipesauthordata">StarlightRecipesAuthorData</a>[]</code>

The list of authors of the recipe.

#### `tags`

**Type:** `{ label: string; href: string; }[]`

The list of tags of the recipe.

Each tag has the following properties:

##### `label`

**Type:** `string`

The label of the tag.

##### `href`

**Type:** `string`

The link to the tag page.

### `authors`

**Type:** <code><a href="#starlightrecipesauthordata">StarlightRecipesAuthorData</a>[]</code>

An unordered list of all the known authors in your recipes.

## `StarlightRecipesAuthorData`

Recipe authors are represented by a `StarlightRecipesAuthorData` object with the following properties:

### `name`

**Type:** `string`

The [name](/configuration/#name) of the author.

### `title`

**Type:** `string | undefined`

The optional [title](/configuration/#title) of the author.

### `url`

**Type:** `string | undefined`

The optional [URL](/configuration/#url) to link the author to.
