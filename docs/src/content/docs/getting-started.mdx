An opinionated [Starlight](https://starlight.astro.build) plugin to create a recipe website.

Starlight Recipes supports the following features:

- Paginated recipe browsing and discovery
- Global and per-recipe author attribution
- High-resolution cover image support
- Custom sidebar with featured content
- Cooking mode: Integrated timers and checkboxes
- Dynamic ingredient scaling by serving size
- Print-optimized layouts for every recipe
- Automated SEO via [recipe structured data](/guides/structured-data)

The following features are available when you allow [on-demand rendering](https://docs.astro.build/en/guides/on-demand-rendering/):

- Automatic [rating system](/interactive/rating-system) based on [Countify](https://countify.xyz/)

Check out the [demo](/recipes) for a preview of an example cookbook.

## Prerequisites

You will need to have a Starlight website set up.
If you don't have one yet, you can follow the ["Getting Started"](https://starlight.astro.build/getting-started) guide in the Starlight docs to create one.

## Installation

import InstallAdapterCommands from '@/components/InstallAdapterCommands.astro'

<Steps>

1. Starlight Recipes is a Starlight [plugin](https://starlight.astro.build/reference/plugins/). Install it by running the following command in your terminal:

   <PackageManagers pkg="starlight-recipes" />

2. Configure the plugin in your Starlight [configuration](https://starlight.astro.build/reference/configuration/#plugins) in the `astro.config.mjs` file.

   ```diff lang="js"
   // astro.config.mjs
   import starlight from '@astrojs/starlight'
   import { defineConfig } from 'astro/config'
   +import starlightRecipes from 'starlight-recipes'

   export default defineConfig({
     integrations: [
       starlight({
   +      plugins: [starlightRecipes()],
         title: 'My Docs',
       }),
     ],
   })
   ```

3. Starlight Recipes uses [Astro's content collections](https://docs.astro.build/en/guides/content-collections/), which are configured in the `src/content.config.ts` file.

    Update the content config file to add support for customizing individual recipes using their frontmatter:

    ```diff lang="ts"
    // src/content.config.ts
    import { defineCollection } from 'astro:content';
    import { docsLoader } from '@astrojs/starlight/loaders';
    import { docsSchema } from '@astrojs/starlight/schema';
    +import { recipesSchema } from 'starlight-recipes/schema'

    export const collections = {
      docs: defineCollection({
        loader: docsLoader(),
        schema: docsSchema({
    +      extend: (context) => recipesSchema(context)
        })
      }),
    };
    ```

    If you are already [extending Starlight’s schema](https://starlight.astro.build/reference/frontmatter/#extend) with additional fields, you can rely on [Zod's API](https://v3.zod.dev/?id=merge) to compose schemas together, e.g. `recipesSchema(context).merge(mySchema)`.

4. Add [Astro's `site` configuration](https://docs.astro.build/en/reference/configuration-reference/#site) to enable [Structured Data](/guides/structured-data).
    This is necessary to generate valid SEO images.

    ```diff lang="js"
    // astro.config.mjs
    import starlight from '@astrojs/starlight'
    import { defineConfig } from 'astro/config'
    import starlightRecipes from 'starlight-recipes'

    export default defineConfig({
    + site: "https://example.com",
      integrations: [
        starlight({
        plugins: [starlightRecipes()],
        title: 'My Docs',
        }),
      ],
    })
    ```

5. Add any [Astro Server Adapter](https://docs.astro.build/en/guides/on-demand-rendering/#server-adapters) to unlock [interactive features](/interactive).

    <InstallAdapterCommands />

6. [Start the development server](https://starlight.astro.build/getting-started/#start-the-development-server) to preview the plugin in action.

</Steps>
